# История изменений DevEvents

## [1.0.1] - 13 января 2026

---

## 🎯 Основные улучшения

### 1. ✅ Автоматическая дедупликация событий

**Проблема:** События дублировались из разных источников (itevents, iteventsrus), требовалось ручное
нажатие кнопки "Очистить дубликаты".

**Решение:**

- Дедупликация теперь запускается **автоматически** при каждом обновлении событий
- Добавлена проверка общих ключевых названий событий (например, "T-Sync Conf")
- Разделены пороги similarity для заголовков (0.65) и описаний (0.55)

**Файлы:**

- `EventRepositoryImpl.kt` - добавлен автоматический вызов `cleanupExistingDuplicates()`
- `TextSimilarity.kt` - метод `hasCommonEventNames()` для поиска общих названий
- `EventDeduplicationService.kt` - обновлена логика с тремя стратегиями
- `TelegramTitleExtractor.kt` - улучшено извлечение заголовков по паттернам

### 2. 📸 Улучшенное извлечение изображений

**Проблема:** События не имели изображений, хотя они были доступны на страницах мероприятий.

**Решение:**

- Добавлено извлечение Open Graph изображений из внешних ссылок
- 4-уровневый fallback: Open Graph → Twitter Card → Link rel → Контент страницы
- Автоматическая замена HTTP на HTTPS
- Фильтрация неподходящих доменов (YouTube, соцсети)
- Timeout увеличен до 10 секунд

**Файлы:**

- `TelegramImageExtractor.kt` - полностью переработан с поддержкой Open Graph

### 3. 📊 Увеличен охват событий

**Проблема:** Парсились только последние 10 сообщений, старые события не попадали в приложение.

**Решение:**

- Лимит парсинга увеличен с 10 до 30 сообщений
- Теперь охватывается примерно неделя активности канала

**Файлы:**

- `TelegramParser.kt` - `PARSE_LIMIT = 30`

### 4. 🛡️ Защита от ошибок при парсинге

**Проблема:** Ошибки при извлечении изображений приводили к потере событий.

**Решение:**

- Все операции извлечения изображений обернуты в `runCatching`
- События создаются даже при ошибках (просто без изображений)
- Graceful degradation для сетевых ошибок

**Файлы:**

- `TelegramMessagePipeline.kt` - защита от ошибок при извлечении изображений

## 📋 Детальные изменения

### Дедупликация

#### EventDeduplicationService.kt

- Разделены константы `TITLE_SIMILARITY_THRESHOLD` и `DESCRIPTION_SIMILARITY_THRESHOLD`
- Добавлена проверка `hasCommonEventNames` в три метода:
    - `removeDuplicatesFromDatabase()`
    - `filterOutDuplicatesOfFavorites()`
    - `cleanupExistingDuplicates()`

#### TextSimilarity.kt

- Добавлен метод `hasCommonEventNames()` для поиска общих названий событий
- Добавлен метод `extractEventNames()` для извлечения названий из текста
- Оптимизирован поиск пересечений (HashSet, ранние выходы)
- Добавлены константы `eventKeywords` для ключевых слов

#### TelegramTitleExtractor.kt

- Добавлен метод `findEventNameByPattern()` для поиска по паттернам
- Поддержка паттернов: "Название Conf", в кавычках, заглавные слова

#### EventRepositoryImpl.kt

- Добавлен автоматический вызов `cleanupExistingDuplicates()` после вставки событий

### Извлечение изображений

#### TelegramImageExtractor.kt

Полностью переработан:

- Константы вынесены в companion object
- Метод разбит на 5 логических частей
- Добавлено извлечение Open Graph, Twitter Card, Link rel
- Добавлен fallback: поиск изображений в контенте страницы
- Автоматическая нормализация URL (HTTP → HTTPS)
- Фильтрация неподходящих доменов
- Упрощено логирование

#### TelegramMessagePipeline.kt

- Извлечение изображений обернуто в `runCatching`
- Упрощено логирование
- Использование функционального стиля

### Парсинг

#### TelegramParser.kt

- `PARSE_LIMIT` увеличен с 10 до 30
- Функциональный стиль вместо императивного
- Упрощено логирование
